# Deterministic Chess Decision Benchmark

This project evaluates language models as constrained decision systems rather than free-form chess generators.

## Protocol lock

- Provider/model identifiers are recorded exactly as submitted.
- Temperature is fixed at `0`.
- Paper-safe mode is `constrained_index`.
- Legal moves are generated by `chess.js`.
- The model selects an integer index only.
- One retry is allowed on invalid output.
- Provider failures are handled separately from model-output failures:
  - rate limits trigger key cooldown, backoff, and retry
  - fallback is not used immediately on provider failure
- Final fallback is deterministic, non-reversing when alternatives exist, and explicitly logged.
- Randomness is controlled by a recorded seed.

## Run outputs

Each run writes:

- `accepted-config.json`
- `manifest.json`
- `run_manifest.json`
- `status.json`
- `pipeline.log`
- `health.json` per matchup
- per-matchup `moves.jsonl`
- per-matchup `games.jsonl`
- per-matchup `paper-results.json`
- per-matchup `paper-stats.json`
- `stats.json`
- `figures_data.json`

## Research claim

The benchmark is designed to separate action hallucination from downstream decision failure. Illegal actions are never executed; they are rejected, retried once, and then replaced by a deterministic fallback.

## Resume protocol

The pipeline checkpoints move-level datapoints and completed games while the run is in progress. If the process restarts, the interrupted matchup resumes from the next unfinished game. This is intentionally a game-boundary resume policy: completed games are preserved, while an in-flight game may be replayed.
